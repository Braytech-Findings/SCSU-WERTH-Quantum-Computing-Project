# Why This Matters to Society

Quantum computers are not all built the same way. A superconducting processor can have fast operations but limited neighbor connections. A trapped-ion processor can offer broad connectivity but has different timing and error tradeoffs. A qubit count alone therefore does not tell a researcher which system fits a circuit.

Fair, transparent comparisons can support education, scientific simulation, chemistry, materials research, optimization, and cryptography research. They can also reduce wasted compute time and provider credits by showing why circuit structure, routing, native gates, and evidence type matter when selecting hardware.

This project contributes reproducible proxy models plus separately labeled provider validation. Students can learn compilation concepts; researchers can audit assumptions; organizations can make more informed experimental plans. The results do **not** prove that one architecture is universally better, predict live-device performance, or show that current quantum computers solve every important problem.

Important uncertainties remain: proxy timing/error assumptions are not calibrations, the benchmark suite is small, IBM physical-hardware evidence is not matched to a physical Quantinuum QPU run, and hardware behavior changes over time. Stronger future work needs preregistered matched circuits, contemporaneous calibrations, repeated physical-QPU runs, uncertainty analysis, cost reporting, and independent replication.
