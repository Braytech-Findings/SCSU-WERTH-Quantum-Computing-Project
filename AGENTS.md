# Project Rules

- Follow the repository plan and keep the implementation scientifically accurate and accessible to advanced high-school students.
- Prefer current official documentation and installed package behavior over outdated examples.
- Never fabricate backend availability, hardware execution, or device-specific noise values.
- Keep the same logical circuits for all execution pipelines.
- Preserve measurement endianness consistently and document it.
- Use null rather than zero for unavailable measurements.
- Do not commit secrets, credentials, or API keys.
- Keep tests and documentation updated alongside implementation changes.
- Do not submit paid hardware jobs or consume credits without explicit approval.
